# Mobile System Design Interview Checklist

* [ ] **Functional Requirements**: Did I clarify what the user can do?
* [ ] **Non Functional Requirements**: Did I discuss performance, reliability, etc.?
* [ ] **Architecture**: High-level block diagram and component interaction.
* [ ] **APIs**: Request/Response models for key features.
* [ ] **Database**: Local storage schema and relations.
* [ ] **Cache**: Strategies for improving latency and offline support.
* [ ] **Offline Support**: Conflict resolution and sync strategy.
* [ ] **Pagination**: Offset-based vs. Cursor-based.
* [ ] **Retry Strategy**: Exponential backoff and idempotency.
* [ ] **Error Handling**: Friendly UI messages and logging.
* [ ] **Performance Optimizations**: Image loading, background sync, resource management.
* [ ] **Security Considerations**: Authentication, TLS, ProGuard/R8, Secure storage.
* [ ] **Trade-offs**: Why did I choose X over Y?
